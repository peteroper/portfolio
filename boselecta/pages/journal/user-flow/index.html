Automated Userflow and Screen Repository for a sprint project

It's easy to skip mapping user flows, but when teams have a shared visual guide, decisions get sharper, and the product comes together more smoothly.

The Challenge

When I joined my current company, I quickly recognized the value of the app’s 200+ screen user flow in helping me get up to speed. However, each sprint cycle required me to manually update the flow, which became a time-consuming and repetitive task. Realizing an opportunity to improve this, I applied my systemized approach to automate the process using auto-layout and a screen repository.

"Userflows aren't glamorous, but many people rely on a userflow for a variety of tasks. When they work well, it's not seen, but when they fail, it's noticed."

The Challenge

User flows were previously updated manually by copying sprint handoff screens into a new version of the flow. Sections had to be dragged into place, and flow lines reconnected each time—making it a laborious and time-consuming task.

This process also exposed a deeper issue: the lack of a single source of truth. Without an original, structured base to work from, tracking new screen versions and ensuring consistency across updates became increasingly difficult. It often led to duplication, version mismatches, and time lost chasing clarity.

Refining the Workflow

I began by componentizing much of the app’s UI—eliminating duplication and using Figma’s library functionality to maintain consistency across variables, text styles, and UI details. This foundation allowed me to scale up into a screen repository, organizing different app sections as reusable components with built-in version control.

Building on that, I started automating the user flow with auto layout. I nested screens from the library into device frames, so whenever I updated a screen, the flow would automatically adjust — even if the screen height changed. This setup meant different parts of the flow could evolve on their own without needing to rebuild everything, making updates much more seamless.

What This Unlocked

  • User flows became more naturally integrated into the final stages of design sprints and handoff files, no longer feeling like a last-minute or reluctant task.
  • Alignment with the development cycle helped reduce friction with the QA team, as complete and up-to-date flows were ready for the UAT phase.
  • For the UX team, the shared screen library became a reliable source of truth, allowing everyone to pull the latest screens into their projects and significantly reducing inconsistencies during design ideation phases.
  • The screen library also made tracking changes across screens much easier, offering a quick, visual way to review progress and updates.

The Invisible Work

Some of the most valuable work in design is also the least visible. Maintaining user flows, managing variables, and keeping systems consistent often go unnoticed—especially when their impact stretches quietly across multiple teams.

Many in the organization rely on these flows for different reasons— Directors and PMs planning strategy, New Joiners learning the system, to QA and development tracking changesBut few see the ongoing effort it takes to keep everything aligned and up to date. By streamlining processes, automating repetitive tasks, and building systems that scale, I’ve helped reduce that hidden load.

The result? Less time spent fixing broken pieces, and more time freed for creative thinking and deeper problem-solving. It’s the quiet, foundational work that holds everything together—and makes the visible work shine.
